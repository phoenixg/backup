Model-View-Controller

BackendPro 是基于MVC开发模式. 更多见 CodeIgniter 和 MVC.

Matchbox 火柴盒

现在你读到用户手册里说 BackendPro 用到了 Matchbox.这是啥玩意儿呢? 正如上面所说，BackendPro 使用了MVC 模式来分离文件. Matchbox 允许了其他独立模块，叫做modules. 用例子来说明如下

CI的应用程序对于小型站点来说有如下的几个文件:

System
    Application
        Controllers
            Home.php
            Widgets.php
            Pages.php
            Contacts.php
        Models
            Widget_model.php
            Page_model.php
            Contacts_model.php
        Views
            Widget_view.php
            Page_view.php
            Contacts_view.php

对于小型站点来说，现在上面的代码已经够用了. 但如果我们有15个控制器和15个模型等等. 那么就很难过了. 于是 matchbox 和它的 modules 允许你把这些文件组成一个组，如下:

System
    Application
        Controllers
            Home.php
        Models
        Views
Modules
    Page
        Controllers
            Page.php
        Models
            Page_model.php
        Views
            Page_view.php
    Widgets
        Controllers
            Widgets.php
        Models
            Widget_model.php
        Views
            Widget_view.php
    Contacts
        Controllers
            Contacts.php
        Models
            Contact_model.php
        Views
            Contact_view.php

但在，也许看起来更复杂了，但是你很容易就知道widget文件在哪儿. A使用modules的另一个巨大优势在于，如果你在其他应用程序里要重用一个部分的话,那么很容易就可以.拿Contact代码来说. 你无需去搜索所有需要required的文件来确保其工作.

Assets

一个asset是一个应用程序使用的有用的文件. 比如 CSS/JS 文件, 图标和图片等等. BackendPro 使用了一个独特的 asset 管理特性，允许所有assets在被加载的时候，开发者都无需担心这是如何实现的。

所有 assets 都位于 Assets 文件夹，如下结构：

Assets
    Admin
        JS
        Images
        CSS
    Public
        JS
        Images
        CSS
    Images
    Shared
        JS
        Icons
        Images
        CSS

你可以把 admin/public 区域的assets分离开来，或者让其共享assets
