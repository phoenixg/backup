JS Compression JS压缩

Asset Library 自带了压缩和优化你的JS代码的能力. 该插件叫做 /Packer/ ,用于执行实际的优化.

在你的asset文件上运行经过压缩后的js的巨大优势是，这表示它会更小，也就意味着更快地下载到你的用户的机器上.

Note: JS 压缩和优化不会更改你服务器上面的asset文件, 它只影响 cached 的副本.

系统要求

  1. 开启Asset优化:

    你必须为asset类库开启asset优化, 更多信息参见 Asset 优化 页面.

  2. 设置 Packer:
    1. 下载最新版本的Packer 下载页面.
    2. 解压zip内容至 modules/site/plugins/packer/.
    3. 打开 modules/site/config/bep_assets.php 配置文件, 找到并设置如下的键值 (大致位于文件的底部): $config['packer']['path'] = BASEPATH . "../modules/site/plugins/packer/class.JavaScriptPacker.php4"; // PHP 4
      $config['packer']['path'] = BASEPATH . "../modules/site/plugins/packer/class.JavaScriptPacker.php"; // PHP 5

现在当用户访问网站时，js文件会经过压缩并且存储.

Note: JS packer only works on JS which has not yet been minimised/packed. If you run it on packed JS it tends to fail.
